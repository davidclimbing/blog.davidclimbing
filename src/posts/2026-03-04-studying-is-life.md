## 브라우저 주소창에 URL 을 입력해서 IP 주소를 받는 일련의 과정?

입력된 URL 을 브라우저가 파싱을 하고, 파싱한 URL 의 도메인을 추출하여 IP 주소를 얻기 위해 여러 단계를 거쳐서 IP 주소를 얻습니다.

1. 브라우저 캐시를 확인하여 도메인의 IP 존재 유무를 확인
2. OS 의 hosts 파일을 확인하여 IP 존재 유무를 확인
3. ISP 의 DNS resolver 를 통해 Root name server 에 도메인의 IP 주소를 요청
4. TLD (Top Level Domain) - .com, .io 와 같은 널리 쓰이는 도메인 서버에 IP 주소를 요청
5. Auth(뭐시기 있는데?) 도메인 서버에 IP 를 요청하여 최종적으로 IP 주소를 받음

## 입력한 도메인의 IP 주소를 받고 나서 브라우저에 화면이 보이기까지 일어나는 이벤트들은?

https 요청을 하기 위해서 우선, TCP 연결을 합니다.

    1. 브라우저에서 서버로 SYN 패킷을 보내고
    2. 서버는 SYN 패킷을 받으면 해당 패킷을 보낸 클라이언트로 SYN-ACK 패킷을 보냅니다.
    3. 클라이언트는 SYN-ACK 패킷을 받으면 마지막으로 ACK 패킷을 서버로 보내고 연결을 완료합니다.

TCP 연결이 완료된 직후, 추가로 브라우저는 TLS 핸드쉐이크를 서버에게 요청합니다. TLS 핸드쉐이크가 완료되면 클라이언트와 서버 간의 대칭키가 완성되어 두 주체간의 통신이 편리해집니다...

이후 서버는 브라우저가 보낸 요청에 응답을 보냅니다. 브라우저는 서버의 응답을 받고 받은 응답 파일들을 파싱합니다. html 파일은 파싱해서 DOM 트리를 생성하고, css 파일은 CSSOM 트리를 생성합니다. 각 2개의 트리를 생성하고 나면 Render Tree 를 생성하여 리플로우와 리페인트 작업을 거쳐 Composition 을 최종으로 화면에 렌더링을 완료합니다.
